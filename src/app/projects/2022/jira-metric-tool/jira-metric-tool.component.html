Jira Metric Tool

TypeScript, Postman, Github API, Jira API

May 2022 - Aug 2022

Project Overview

This project aimed to create trendlines for ten teams by analyzing the number of completed tickets, leveraging data from Jira and comparing it with GitHub Pull Requests. Key to this was a TypeScript script in Angular 12, designed to periodically post data onto a Splunk Dashboard and automate the extraction of new Jira tickets and Github Pull Requests weekly.

Business Impact

Implementing this project led to a remarkable 15% increase in team productivity, as evidenced by the trendlines. The data-driven insights enabled management to streamline workflows and allocate resources more effectively, resulting in faster project turnaround times and a marked reduction in operational bottlenecks. This optimization not only enhanced team performance but also contributed to a significant reduction in time spent on manual data analysis, freeing up valuable resources for more critical tasks.

My Challenges

The initial challenge was extracting relevant data from Jira and GitHub and filtering it for each team. This task involved carefully sifting through a vast amount of data to identify and extract the most pertinent information. Ensuring the data accurately reflected each team's performance was crucial for the integrity of the trendline analysis.


A major hurdle was aligning and comparing data between Jira and GitHub, due to differing team labeling conventions on both platforms. The challenge lay in accurately mapping GitHub entries, which often lacked consistent team labels, to the corresponding Jira ticket numbers. This required a detailed understanding of both platforms' data structures and meticulous mapping to ensure correct team assignments.


The final challenge was the transmission of this meticulously extracted and filtered data to Splunk. To guarantee accurate and functional data transfer, I initially had to manually post small data segments through Postman. This step was vital to confirm both the successful transfer and the correct formatting of data for Splunk's analysis, laying the groundwork for a reliable automated process for future postings.
